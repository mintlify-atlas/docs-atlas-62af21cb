# Bitácora Operativa

Bitácora Operativa is a full-stack web application designed for operations teams to log, track, and publish operational events. It integrates with NMS/EMS inventory systems, supports SAP-like technical locations and equipment hierarchies, and provides a complete audit trail with role-based access control.

## Key capabilities

- **Draft/publish workflow** — entries start as drafts and are published explicitly by operators or administrators
- **Role-based access** — three access levels: public read, operator, and administrator
- **Rich attachments** — WYSIWYG editor with inline images (normalized to PNG) and file attachments
- **External references** — link entries to external tickets/folios validated by configurable regex patterns
- **CSV import rules** — define and execute import rules to load NMS/EMS inventory data from CSV files
- **Rate limiting** — configurable per-environment API and login rate limits
- **Configurable public access** — the public timeline can be toggled on/off from system configuration

## Tech stack

| Layer | Technology |
|---|---|
| Backend | Laravel 10 (PHP 8.1+) |
| Frontend | Vue 3 + Vite (SPA) |
| Database | MySQL 8 |
| Authentication | Laravel Sanctum (session + CSRF) |
| Authorization | Spatie Roles/Permissions |
| Editor | Tiptap (WYSIWYG) |
| Charts | Chart.js |

## User roles

| Role | Capabilities |
|---|---|
| **Public** | Read published entries, view catalogs, view inventory elements |
| **Operator** | All public + create/edit/delete entries, upload attachments, manage own profile |
| **Administrator** | All operator + manage catalogs, PM matrix, inventory, users, system configuration |
