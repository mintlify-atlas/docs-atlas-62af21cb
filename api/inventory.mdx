---
title: "Inventory elements (public)"
description: "List active network inventory elements including nodes, links, services, and tunnels."
---

The inventory endpoint exposes active network elements from the operational inventory. No authentication is required.

---

## List inventory elements

`GET /api/v1/inventario/elementos`

Returns a flat (non-paginated) array of active `InvElementoRed` records ordered by `codigo`. Each element includes its parent network (`red`), domain list, and type-specific detail fields.

### Query parameters

<ParamField query="red_id" type="integer">
  Filter elements belonging to a specific network ID.
</ParamField>

<ParamField query="q" type="string">
  Search term. Matches against `codigo`, `nombre`, and `tipo` using a partial (`LIKE`) match.
</ParamField>

<ParamField query="limit" type="integer" default="(none)">
  Maximum number of records to return. Capped at `1000`.
</ParamField>

### Example request

```bash
curl -X GET "https://your-app.example.com/api/v1/inventario/elementos?q=NORTE&limit=10" \
  -H "Accept: application/json"
```
