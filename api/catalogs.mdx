---
title: "Catalogs (public)"
description: "Read-only catalog endpoints for entry criteria, impact levels, and PM work order/activity classes."
---

These endpoints return the active catalog values used when classifying log entries. No authentication is required.

---

## Entry criteria

`GET /api/v1/catalogos/criterios`

Returns all active entry criterion records, ordered by `orden`.

### Response

<ResponseField name="data" type="object[]">
  <Expandable title="CatEntradaCriterio">
    <ResponseField name="id" type="integer">Record ID.</ResponseField>
    <ResponseField name="codigo" type="string">Short code (e.g. `INT`).</ResponseField>
    <ResponseField name="nombre" type="string">Display name.</ResponseField>
    <ResponseField name="descripcion" type="string | null">Optional description.</ResponseField>
    <ResponseField name="orden" type="integer">Sort order.</ResponseField>
    <ResponseField name="activo" type="boolean">Whether the criterion is active.</ResponseField>
  </Expandable>
</ResponseField>

### Example request

```bash
curl -X GET "https://your-app.example.com/api/v1/catalogos/criterios" \
  -H "Accept: application/json"
```

### Example response

```json
{
  "data": [
    {
      "id": 1,
      "codigo": "MAN",
      "nombre": "Mantenimiento programado",
      "descripcion": "Trabajos de mantenimiento preventivo calendarizados.",
      "orden": 1,
      "activo": true
    },
    {
      "id": 2,
      "codigo": "INT",
      "nombre": "Interrupción de servicio",
      "descripcion": null,
      "orden": 2,
      "activo": true
    }
  ]
}
```

## PM order-activity matrix

`GET /api/v1/catalogos/pm/matriz`

Returns the valid combinations of PM work order classes and activity classes. When creating an entry you must use a combination present in this matrix.

### Response

<ResponseField name="data" type="object[]">
  <Expandable title="PmMatrizOrdenActividad">
    <ResponseField name="id" type="integer">Record ID.</ResponseField>
    <ResponseField name="pm_clase_orden_id" type="integer">ID of the work order class.</ResponseField>
    <ResponseField name="pm_clase_actividad_id" type="integer">ID of the activity class.</ResponseField>
    <ResponseField name="activo" type="boolean">Whether the combination is active.</ResponseField>
  </Expandable>
</ResponseField>

<Note>
  Cross-reference `pm_clase_orden_id` and `pm_clase_actividad_id` with the `/pm/ordenes` and `/pm/actividades` endpoints to get the human-readable names.
</Note>

### Example request

```bash
curl -X GET "https://your-app.example.com/api/v1/catalogos/pm/matriz" \
  -H "Accept: application/json"
```

### Example response

```json
{
  "data": [
    {
      "id": 1,
      "pm_clase_orden_id": 1,
      "pm_clase_actividad_id": 2,
      "activo": true
    },
    {
      "id": 2,
      "pm_clase_orden_id": 2,
      "pm_clase_actividad_id": 1,
      "activo": true
    }
  ]
}
```
