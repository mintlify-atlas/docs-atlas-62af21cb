## Prerequisites

Before you start, make sure you have the following installed:

- **PHP 8.1+** with the extensions listed in the [deployment pre-checklist](/deployment#pre-checklist)
- **Composer**
- **Node.js 20+**
- **MySQL 8**

## Development setup

<Steps>
  <Step title="Copy the environment file">
    ```bash
    cp .env.example .env
    ```
  </Step>

  <Step title="Configure your database">
    Open `.env` and set your MySQL connection details:

    ```env
    DB_HOST=127.0.0.1
    DB_PORT=3306
    DB_DATABASE=bitacora
    DB_USERNAME=bitacora
    DB_PASSWORD=bitacora
    ```
  </Step>

  <Step title="Install dependencies">
    <CodeGroup>

    ```bash PHP
    composer install
    ```

    ```bash Node
    npm install
    ```

    </CodeGroup>
  </Step>

  <Step title="Generate the application key">
    ```bash
    php artisan key:generate
    ```

    This writes `APP_KEY` to your `.env` file.
  </Step>

  <Step title="Run migrations and seeders">
    ```bash
    php artisan migrate --seed
    ```

    This creates all 12 migration groups and loads the base data seeders:
    `ConfiguracionSistemaSeeder`, `CatalogoOperativoSeeder`, `PmLikeSeeder`,
    `SistemasExternosSeeder`, `InventarioRedesDominiosSeeder`, and `ImportRuleSeeder`.
  </Step>

  <Step title="Create the storage symlink">
    ```bash
    php artisan storage:link
    ```

    This makes `storage/app/public` accessible at `public/storage`.
  </Step>

  <Step title="Start the backend">
    ```bash
    php artisan serve --host=0.0.0.0 --port=8000
    ```

    The API and SPA shell are now available at `http://localhost:8000`.
  </Step>

  <Step title="Start the frontend">
    Open a second terminal and run:

    ```bash
    npm run dev -- --host 127.0.0.1 --port 5173
    ```

    Vite serves the Vue SPA with hot-module replacement on port `5173`.
    Open your browser at `http://127.0.0.1:5173`.
  </Step>
</Steps>

## Default seed users

The `ConfiguracionSistemaSeeder` creates two users with password `12345678`:

| Email | Role |
|---|---|
| `admin@bitacora.test` | `administrador` |
| `operador@bitacora.test` | `operador` |

<Warning>
  Change both default passwords immediately after your first login. The default password `12345678` must never be used in a production environment.
</Warning>

## First login

Navigate to `/login`, sign in with one of the seed accounts, then go to **Configuración** to update your password.

## Optional: Podman + Makefile

If you prefer to run MySQL in a container, a `compose.yml` and `Makefile` are included for use with Podman Compose:

<CodeGroup>

```bash Start MySQL container
make up
```

```bash Start Laravel backend
make serve
```

```bash Start Vite frontend
make dev
```

</CodeGroup>

The `compose.yml` starts a `mysql:8.0` container named `bitacora_mysql` with database `bitacora`, user `bitacora`, and password `bitacora` on port `3306`.

<Note>
  `make up` uses `podman-compose`. If you use Docker Compose, run `docker compose up -d` directly.
</Note>

## Useful commands

| Command | Purpose |
|---|---|
| `php artisan migrate:fresh --seed` | Drop all tables and re-seed from scratch |
| `php artisan optimize:clear` | Clear all cached config, routes, and views |
| `npm run build` | Build production assets |
